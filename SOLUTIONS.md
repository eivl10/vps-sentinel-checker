# База знаний и проверенные решения (Solutions & Lessons Learned)

Данный документ содержит зафиксированные практические решения и правила для автономного мониторинга VPS и управления Telegram-ботом.

---

## 1. Авторизация новых пользователей в 1 клик (Approval Workflow)

> **Проблема:** Добавленный по `@username` новый пользователь не мог войти в бот, так как Telegram API авторизует строго по числовому `chat_id`.
> 
> **Решение:**
> - Когда новый пользователь пишет боту `/start`, бот НЕ блокирует его всухую, а формирует инлайн-запрос Главной Администратору (ID: `1447443197`):
>   - `📥 Запрос на доступ от @username (ID: 6694926660)`
>   - Кнопки: `[Одобрить: Базовый (Партнёрка+ЦЖ)]`, `[Одобрить: Полный (Все сервисы)]`, `[Отклонить]`.
> - Нажатие на любую кнопку моментально выдаёт выбранный уровень доступа с автоматическим обновлением `config.json`.

---

## 2. Уровни доступа (Базовый vs Полный доступ)

> **Задача:** Одним пользователям нужно видеть только **Партнёрский кабинет** и **ЦЖ**, а другим — **Все сервисы** (включая Курсы валют и Утку).
> 
> **Решение:**
> - Добавлены роли пользователей:
>   - **`base` (Базовый)**: Видит только *Партнёрский кабинет* и *ЦЖ*.
>   - **`full` (Полный)**: Видит все 4 группы (*Партнёрка*, *ЦЖ*, *Курсы*, *Утка*).
>   - **`admin` (Администратор)**: Полный доступ + управление доступом и ролями.
> - В разделе `Пользователи бота` у Администратора доступны кнопки мгновенной смены роли любого пользователя (`[Сменить доступ на: Полный / Базовый]`).

## 3. Умный перезапуск процессов без ошибок

> **Решение:**
> - Для сервисов PM2 (`cabinet-backend`, `vps-sentinel`) используется `pm2 restart`.
> - Для веб-серверов используется `systemctl restart nginx`.
> - Для Python-процессов проверяется текущее состояние `ps aux`. Если процесс уже активен, бот возвращает корректный статус `Работает штатно (активен)` без вызова принудительных ошибок.

---

## 4. Гибридная проверка процессов (PM2 + Nginx + PostgreSQL + Python)

> **Решение:**
> - Использовать **гибридную систему инспектирования**:
>   - **PM2**: Вызов `pm2 jlist` для Node.js/PM2 сервисов.
>   - **HTTP & TCP порты**: Опрос портов `80` (Nginx), `3000` (Бэкенд API), `5432` (PostgreSQL).
>   - **Системные процессы (`ps aux`)**: Сканирование активных процессов Python (`kurrsator_bot.py`, `main.py`, `telegram_bot`, `transutka-qwen`).

---

## 5. Бессмертное развёртывание на VPS через PM2

> **Решение:**
> - Бот разворачивается в PM2: `pm2 start /home/antigravity/vps-monitor/vps_monitor.py --name vps-sentinel --interpreter python3`.
> - Команда `pm2 save` фиксирует состояние процессов в случае перезагрузки VPS.
> - Потребление ресурсов: **< 15 MB RAM**, **0% CPU**.
